# Лабораторная работа №4

## Тема лабораторной работы
Иерархические структуры данных. Красно-черные деревья (**23 вариант**).

## Цель работы
Составить программу на языке Си для реализации динамической самобалансирующейся структуры данных. Программа должна обеспечивать выполнение операций вставки, удаления и поиска за время $O(\log n)$, поддерживая балансировку дерева через систему правил и поворотов.

## Теоретические сведения

### Основные понятия
Красно-черное дерево — это вид бинарного дерева поиска, в котором каждый узел имеет дополнительный атрибут — **цвет** (красный или черный). Оно гарантирует, что дерево остается сбалансированным, предотвращая его вырождение в список.



### 5 обязательных свойств
Для поддержания баланса должны соблюдаться следующие правила:
1.  **Цвет:** Каждый узел либо красный, либо черный.
2.  **Корень:** Всегда черный.
3.  **Листья (NIL):** Все фиктивные листья (NIL-узлы) считаются черными.
4.  **Красные узлы:** Красный узел не может иметь красных детей (отсутствие двух красных узлов подряд).
5.  **Черная высота:** Любой путь от узла до его листьев содержит одинаковое количество черных узлов.

### Механизмы балансировки
Если вставка или удаление нарушают правила, дерево восстанавливается с помощью:
* **Перекрашивания:** Изменения цветов узлов в зависимости от ситуации.
* **Поворотов:** Локальных преобразований структуры (Left/Right Rotate), которые изменяют высоту поддеревьев.

## Идея, метод, алгоритм решения задачи
Центральной частью реализации является обработка случаев нарушения баланса.

**Ключевые алгоритмы:**
1.  **Использование NIL-узла:** Вместо проверок на `NULL` используется единый глобальный объект `tree->NIL` черного цвета, что делает код более чистым и безопасным.
2.  **Вставка (`insert_fixup`):** После вставки нового красного узла алгоритм проверяет цвет «дяди». Если дядя красный — выполняется перекрашивание. Если черный — выполняются повороты.
3.  **Удаление (`rb_delete`):** Реализовано через поиск минимального элемента в правом поддереве для замены и последующий вызов `delete_fixup`, если удаленный узел был черным.

## Характеристики реализации
- **Данные:** Ключи — строки (до 6 символов), значения — `double`.
- **Обработка ключей:** Программа автоматически обрезает входящие строки до 6 символов при помощи `strncpy` для соблюдения лимитов памяти.
- **Режимы:** Поддержка интерактивного меню и пакетной обработки файлов.

## Вывод
В ходе работы реализовано красно-чёрное дерево на C, обеспечивающее вставку, удаление и поиск за $O(log n)$. Балансировка поддерживается автоматически за счёт локальных поворотов и перекрашивания узлов при соблюдении пяти свойств КЧД. В коде применён NIL для безопасной обработки граничных случаев, реализовано корректное управление динамической памятью и поддержка консольного/файлового режимов. Автоматические тесты подтвердили целостность структуры BST, работоспособность алгоритмов балансировки и отсутствие утечек памяти. Цель лабораторной работы полностью достигнута.
